Controlled Benchmark From Existing Pilot Runs

E001-E008 show the model is sensitive to character-pose entanglement, skeleton hints, viewpoint, and pose difficulty. This screen compresses those observations into a first controlled benchmark shape.

What The Pilot Already Tested

1

Rest-pose character source helps

E006 and E007 are cleaner than earlier posed-character inputs because the character image carries less pose interference.

2

No-skeleton references are useful

Removing skeleton lines reduced some confusion, but did not eliminate left-right flips or limb artifacts.

3

Harder poses expose the core failure

E008 with a larger pose change produced consistent limb-side errors across models, so the benchmark needs pose difficulty levels.

4

Prompt should be fixed

E005-E008 already converged to a strong role/view/left-right prompt. Future tests should vary inputs, not rewrite the prompt every time.
